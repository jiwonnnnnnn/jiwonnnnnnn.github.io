1. Sementic segmentaion

  • 주어진 데이터가 차량의 일부 이미지와 그에 해당하는 mask로 구성 따라서 task를 segmentation으로 결정

2. Resnet152, Eff-Unet

  • Resnet152
    • 실제 production을 고려해 본다면 소비자로부터 전달 받은 모든 이미지를 Local Damage Detection(segmentation model)에 넣는 것은 불필요하다고 판단하여 Global damage classifier (classification model, resnet152)를 사용하여 1차적인 검수를 진행하여 비용 절감 및 시간 감소
    • classification model 특성상 test time 차이가 production level에서 영향이 미미하다고 판단하여 깊지만 성능이 잘나오는 resnet152로 결정

  • Eff-Unet
    • 여러가지 sementic segmentaion model들중 컴퓨터성능과 모델 성능를 비교하여 후보를 결정 (eff-unet, deeplabv3)
    • 이후 각 모델의 성능을 비교하여 최종모델로 eff-unetB4로 결정

3. Metric

  • 차량파손모델 특성상 모델이 잘 못 판단하는 경우 파손이 안 된 차량을 파손이 됐다고 판단하는 것보다 파손이 된 차량을 파손 안 됐다고 판단하는 것이 낫다고 생각 따라서 recall을 사용
  • 하지만 recall만을 사용하기에는 모델의 성능이 저하 될 우려가 있어 주된 metric으로 recall을 사용하되 어느 정도의 IoU, dice, precision을 고려

4. Semi-supervised learning

  • 주어진 이미지 데이터는 크게 2개의 데이터셋으로 구성, 즉 하나의 이미지에 대해 3개의 class mask가 존재하지 않고 2개 혹은 1개의 class mask가 존재 따라서 segmentaion model을 3개를 써야함
  • 하지만 3개의 모델을 사용하는 것은 비효율적이라 판단
  • 따라서 하나의 이미지에 대해 존재하지 않는 class의 pseudo mask를 만드는 teacher model과 pseudo mask와 GT mask를 혼합하여 학습하는 student model로 분리하여 1개의 모델로 전체 데이터를 학습 가능
  • test time에는 student model만 사용하기 때문에 production level에서 시간 감소

5. Multi-loss function

Grad-CAM loss

  • Attention기법과 같이 파손부위를 강조하기 위해 Global Damage Detector에서 multi-label classification을 진행하여 3개의 class mask에 대한 class activation map(CAM)을 추출
  • 추출한 CAM과 prediction의 loss를 계산

BCE + Focal loss

  • 각 class별 mask loss를 계산하기 위해 Binary cross entropy를 사용, 각각의 이미지에 픽셀 단위 negative와 positive간의 class imbalance를 해결하기 위해 Focal loss를 적용
  • BCE loss만을 사용하면 recall값은 높지만, 나머지 평가지표의 결과들이 전반적으로 낮음
  • Focal loss만을 사용하면 매우 작은 loss가 발생하여 gradient의 update step이 낮게 발생
  • 따라서 Focal loss로 easy example의 weight를 줄이고 BCE loss의 pos weight로 hard negative의 절대적인 loss를 증가시켜 학습의 반영되는 gradient를 높임

  1. Torchserve

1. Torchserve

  • Google Cloud Platform의 VM위에 docker를 띄운 뒤 torchserve로 model을 serving함
  • model server는 request를 받으면 이미지를 GDC(Global Damage Classification)를 통과 시킨 후 metric 기준에 통과한 이미지들만 LDD(Local Damage Detector)에 통과시킨다.
  • 그 후 모델에서 나온 최종 prediction을 GCS(Google Cloud Storage)에 저장

2. Google Cloud Function

  • 사용자가 6장의 차량 이미지를 올리면 GCS(Google Cloud Storage)에 저장 되며 GSC에 request가 오면 GCF(Google Cloud Function) 트릭이 발도됨
  • GCF는 6장의 차량 이미지를 AI server에 requset를 보내고 이후 AI server에서 받은 최종 prediction을 DB에 저장함
