소개Computer Vision을 도입하여 자동으로 재활용을 분류해주는 서비스 입니다.

프로젝트 개요

✅ 개발 동기

  • 전 세계적으로 플라스틱 사용량이 증가하고 있으며 최근에 covid-19의 영향으로 더 많은 재활용 쓰레기들이 발생하고 있는 추세
  • 이렇게 많은 재활용 쓰레기들을 처리하기 위해 많은 나라들이 정책을 도입하였고 그 중 독일은 무인회수기에 플라스틱, 유리, 캔 등 다양한 재활용 쓰레기를 넣으면 돈으로 환급해주는 pfand 제도를 시행하고 있음
  • 이와 비슷하게 우리나라에서도 제로 서울, 제로 카페 등 많은 정책을 시행 중이며 특히 독일의 pfand와 유사하게 재활용품을 수거해 포인트로 전환하는 네프론이라는 사업을 시행 중임

✅ 개발 아이템

  • 한국의 네프론과 독일의 Pfand에는 단점이 있는데 바로 네프론은 플라스틱, 캔만 회수가 가능하며 Pfand는 인증마크가 있는 것만 회수가 가능
  • 따라서 이 두 개의 서비스를 합치고 확장해 인증마크없이 분류가 가능하며 플라스틱, 캔 뿐만아니라 다른 재활용품도 회수 가능한 서비스를 제공하고자 함

✅ 기대 효과

  • 우리의 재활용 분류 서비스 모델의 목표는 더 많은 재활용 쓰레기를 회수 할 뿐만아니라 소비자들이 경제적인 이득을 얻어 더욱 재활용 쓰레기 처리에 관심을 끌 수 있도록 하는 것임

프로젝트 내용

1. Data part

  • 사용한 데이터 : TACO Dataset
  • 데이터 설명
    • TACO dataset은 야생 폐기물의 공개 이미지 데이터이며 열대 해변에서 런던 거리에 이르기까지 다양한 환경에서 찍은 쓰레기 데이터가 포함되어 있다.
    • Image : 1500장(train, test, validation)
    • Annotation : category, segmentation area, bounding box, object area
    • Format : COCO annotation
    • Class : super categories 28가지, sub categories 60가지 존재

2. Model part

✅ 연구방향 설정

  • 연구 방향을 Semantic Segmentation으로 진행하여 Base Model을 U-Net으로 설정
  • 하지만 Semantic Segmentation Task는 픽셀 자체가 어떤 Class에 속하는지만 구분가능
  • 기획한 재활용 분류 모델은 같은 Class에 속하는 재활용품 내에서도 구분이 필요하므로 적합하지 않다고 판단하여 연구 방향을 Instance segmentation으로 변경
  • Instance segmentation의 기초인 Mask R-CNN을 Base Model로 설정

✅ 모델링

  • Torchvision의 오픈소스 라이브러리를 사용
    • Train
      • Input : Image, Target
      • Output : Classification Loss, Mask Loss, Bounding Box Regression Loss, Objectness Loss
    • Test
      • Input : Image
      • Output : Bounding Box, Confidence Score, Labels, Masks

✅ 학습 및 평가

  • 사용 툴
    • Code : Visual Studio Code
    • GPU : Colab pro
    • Visualization : Weight and Bias
  • Train 과정
    1. 데이터(Image, Target)를 불러옴
    1. 모델(Mask R-CNN)에 데이터를 넣고 각각의 Loss들을 추출
    1. Loss들의 총합인 Total Loss를 구해 파라미터들을 update
    1. 학습 시 각각의 Loss들을 WandB 라이브러리를 이용해 실시간으로 시각화
    1. 위에 과정들을 반복하여 하이퍼파라미터 튜닝을 진행

  • Test 과정
    1. 데이터(Image)를 불러온다.
    1. 학습된 모델(Mask R-CNN)에 데이터를 넣고 output을 추출
    1. 추출된 output에서 bbox와 mask들을 원본 이미지에 투영시켜 시각화

✅ 문제점과 해결책

  1. Class Imbalance
    • TACO Dataset의 60가지 Sub Categories만을 사용했으나 위의 표와 같이 Sub Catecories는 데이터 불균형 문제가 심각함
    • 따라서 이 60가지의 카테고리를 서비스 차원을 고려하여 분리수거 여부에 따라 8가지의 카테고리로 재분류
    • 병합은 Annotation csv파일을 만들고 데이터가 불러올 때 자동으로 변경되게 함

  1. Resource problem
    • 지원이 많이 되지 않는 프로젝트였기 때문에 GPU, 메모리 등 하드웨어적인 부분에서 문제점 존재
    1. Fine tuning
      • Backbone Network로 imagenet으로 Pretrain된 Resnet50+fpn을 가져온뒤 freeze 시키고 RPN, Classification head, Bounding Box Regression head, Mask Head들만 학습을 진행
    1. Resize Image resolution
      • 제한된 메모리에 데이터를 학습시키기 위해 이미지 사이즈를 400x400의 크기로 줄여 학습을 진행
